The row that tells the story

The sort-order bug carries a trap: a stale comment says event ids are ULIDs (“id order is arrival order”), but the id generator quietly went random two versions ago. Text search finds the comment and believes it.

On haiku, pi grepped, read, trusted the comment, and shipped a wrong fix after 35 steps, $0.183, 2m 45s. Empryo’s genome injects what the code actually does — symbols, calls, real relationships — and haiku landed the correct fix in 10 steps, $0.039, 20s. Comments lie. Graphs don’t.

Caveats

Single trial per cell — agent runs are stochastic, so treat small per-cell deltas as noise (the aggregate direction is consistent across all nine cells). Costs are each agent’s self-reported accounting; token counts are recorded raw. pi could not run opus 4.8 out of the box (model registry tops out at 4-7; its default thinking config is rejected by the 4.8 API) — its opus tier ran with a custom model id and thinking off, the only working configuration, and the failed launches are excluded from pi’s totals. Both agents failed csv-quoting on opus with non-RFC escaping that haiku and sonnet got right. Empryo runs include a 1–2s no-LLM genome pre-warm. Reproduce: bun bench/vs/run-vs.ts.
